Detailed Description

This Lock Backend stores all its data in the filesystem in separate file per node.

This Lock Manager is now deprecated. It has a bug that allows parent collections to be deletes when children deeper in the tree are locked.

This also means that using this backend means you will not pass the Neon Litmus test.

You are recommended to use either the PDO or the File backend instead.

Returns a list of Sabre objects

This method should return all the locks for a particular uri, including locks that might be set on a parent uri.

If returnChildLocks is set to true, this method should also look for any locks in the subtree of the uri for locks.
